# **Default Arguments**

## 1. Introduction

**What is this topic?**  
Default arguments allow function parameters to have predefined values that are used when the caller does not provide them.

**Why does Python have it?**  
They make functions more convenient and flexible, reducing boilerplate when calling functions with common configurations, while maintaining backward compatibility as APIs evolve.

**What problem does it solve?**  
It solves the need for multiple function overloads or verbose calls with repetitive arguments. This is especially valuable in configuration-heavy domains like ML pipelines, data processing, and production services.

---

## 2. Intuition

Imagine ordering pizza:  
- Base parameters: `size="medium", crust="regular"` (defaults).  
- You only specify what you want to change: `pizza("large", toppings=["pepperoni"])`.

Defaults provide sensible starting points while allowing customization.

---

## 3. Syntax

```python
def parse_resume(
    text: str,
    mode: str = "hybrid",                    # Simple default
    confidence_threshold: float = 0.75,
    enabled_sections: list[str] | None = None,  # Careful with mutables
    llm_provider: str = "openai", 
    max_tokens: int = 2000
) -> ParseResult:
    """Parse resume with sensible defaults."""
    if enabled_sections is None:
        enabled_sections = ["skills", "experience", "education"]
    ...
```

**Advanced syntax (Python 3.13+):**
- Default expressions evaluated once at definition time.
- Keyword-only defaults: `def func(*, option: bool = True)`
- Type hints with defaults.

---

## 4. Internal Working

Default values are evaluated **once** when the `def` statement executes and stored in the function’s `__defaults__` tuple.

- Immutable defaults (numbers, strings, tuples, `None`) are safe.
- Mutable defaults (lists, dicts, sets) are shared across all calls → dangerous side effects.
- When calling, Python fills missing arguments from the defaults tuple.

This one-time evaluation is efficient but leads to the famous mutable default gotcha.

---

## 5. Built-in Methods / Operations

- `func.__defaults__` — tuple of default values.
- `inspect.signature(func).parameters` — detailed parameter info including defaults.
- `functools.partial()` — creates new callable with some defaults frozen.

---

## 6. Common Mistakes

1. **Mutable default arguments** (the #1 gotcha):
   ```python
   def bad(items: list = []):  # Shared list!
       items.append(1)
       return items
   ```

2. Using expensive computations as defaults (e.g., loading a model).

3. Changing defaults in a way that breaks existing callers.

4. Relying on defaults for critical validation.

5. Order of defaults — cannot have non-default after default.

---

## 7. Best Practices

- Use `None` as default for mutable types, then initialize inside.
- Place defaults at the end of the parameter list.
- Make defaults cheap and immutable.
- Document defaults clearly in docstring.
- Use keyword-only parameters for most defaults.
- Consider dataclasses or `pydantic` settings for complex default configurations.
- Be conservative when changing defaults in public APIs.

---

## 8. Production Usage

- **API endpoints**: Default pagination, filtering, and output formats.
- **ML training scripts**: Default learning rates, batch sizes, epochs.
- **CLI tools**: Sensible defaults for verbosity, output paths.
- **Libraries**: Allow users to override behavior without subclassing.
- **Configuration layers**: Functions accept `config: Config = DEFAULT_CONFIG`.

---

## 9. Machine Learning Usage

- **Scikit-learn**: Many estimators have extensive defaults (e.g., `random_state=None`).
- **PyTorch optimizers**: `lr=0.001`, `weight_decay=0.0`.
- **Hugging Face**: Model loading functions with defaults for `device`, `torch_dtype`.
- **Data loaders**: `batch_size=32`, `shuffle=True`.
- **LLM calls**: `temperature=0.7`, `max_tokens=1024` — perfect for Resume Intelligence.

## 11. Complexity Analysis

- Evaluation of defaults: O(1) at definition time.
- Function call with defaults: Negligible overhead.
- Mutable defaults can cause O(n) bugs that are hard to trace.
