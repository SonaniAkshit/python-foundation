**Topic**

**Conditional Statements (if, elif, else)**

---

## 1. Introduction

Conditional statements are the fundamental decision-making construct in programming. In Python, the `if`, `elif`, and `else` keywords allow your code to execute different blocks based on whether certain conditions evaluate to `True` or `False`.

Python has them because real-world software must respond dynamically to varying inputs, states, and events. They solve the core problem of **branching logic** — without them, every program would follow a single linear path, making it impossible to build anything useful.

---

## 2. Intuition

Think of an `if` statement like a **bouncer at a club**:
- The bouncer checks ID (the condition).
- If age ≥ 21 → let them in (execute the block).
- Else if age ≥ 18 but has special pass → let them in with restrictions.
- Else → deny entry.

Your code becomes an intelligent agent that can adapt instead of blindly following instructions.

---

## 3. Syntax

```python
# Basic if
if condition:
    # indented block (4 spaces - PEP 8)
    statement1
    statement2

# if-elif-else chain
if condition1:
    ...
elif condition2:      # "else if"
    ...
elif condition3:
    ...
else:
    ...               # optional fallback
```

**Important rules**:
- Colon `:` after every condition.
- Indentation defines the block (no braces `{}` like other languages).
- Conditions must evaluate to `bool` (truthy/falsy values are allowed but be careful).

**One-line version** (use sparingly):
```python
result = "adult" if age >= 18 else "minor"
```

**Chained comparisons** (Pythonic and efficient):
```python
if 18 <= age < 65:
    print("Working age")
```

---

## 4. Internal Working

Python evaluates the condition using its truthiness rules, then jumps to the matching block. Only **one** branch executes per `if-elif-else` chain — this is efficient.

Under the hood, conditions are evaluated left-to-right with short-circuiting (`and`/`or`). The interpreter uses the `PyBool_FromLong` mechanism internally for boolean results.

Memory-wise, each block is just a code object; no extra overhead beyond the evaluation of the expression.

## 5. Built-in Methods / Operations

Key operators used with `if`:

| Operator | Purpose | Example | Time Complexity |
|----------|--------|--------|-----------------|
| `==`, `!=`, `>`, `<`, `>=`, `<=` | Comparison | `if score >= 90` | O(1) |
| `in`, `not in` | Membership | `if "python" in skills` | O(n) for lists |
| `and`, `or`, `not` | Logical | `if active and verified` | Short-circuit |
| `is`, `is not` | Identity | `if result is None` | O(1) |

**Practical example**:
```python
def process_resume_score(score: float, skills: list[str]) -> str:
    if not isinstance(score, (int, float)) or score < 0:
        raise ValueError("Invalid score")
    
    if score >= 90 and "Python" in skills and "ML" in skills:
        return "Senior ML Engineer"
    elif score >= 75:
        return "Mid-level Engineer"
    else:
        return "Junior / Needs improvement"
```

## 6. Common Mistakes

**Mistake 1**: Using assignment `=` instead of comparison `==`
```python
if x = 5:   # SyntaxError or unintended assignment
```

**Mistake 2**: Checking `len(list) > 0` instead of truthiness
```python
if len(skills) > 0:        # Less Pythonic
if skills:                 # Better
```

**Mistake 3**: Deeply nested `if` statements (arrow code) — leads to unreadable, hard-to-test code.

**Correct approach**: Use early returns, guard clauses, or extract helper functions.

---

## 7. Best Practices

- Keep conditions simple and readable.
- Prefer positive conditions (`if active:`) over negative (`if not inactive:`).
- Use `elif` chains instead of nested `if`s when possible.
- Return early to reduce nesting.
- Use type hints and descriptive variable names.
- Avoid magic numbers — use constants or enums.
