# ✅ Practice Tasks for Conditional Logic in Python

Complete these hands-on tasks to solidify your understanding of `if-else`, nested conditionals, `match-case`, and logical problem solving.

---

## 🔘 If-Else

- [ ] **Task 1: Temperature classifier**  
  Write a program that takes a temperature in Celsius and prints:
  - `"Freezing"` if below 0
  - `"Cold"` if between 0 and 10 (inclusive)
  - `"Warm"` if between 11 and 25
  - `"Hot"` if above 25  
  *Use only `if-elif-else` (no nesting).*

- [ ] **Task 2: Leap year checker**  
  Ask the user for a year. Print `"Leap year"` if divisible by 4, but not by 100 unless also divisible by 400. Otherwise print `"Not a leap year"`.  
  *Hint: use the modulo operator `%` and logical operators.*

- [ ] **Task 3: Ternary challenge**  
  Rewrite this block as a single ternary expression:  
  ```python
  if score >= 60:
      result = "Pass"
  else:
      result = "Fail"
  ```
  Then extend it: if score >= 90 → "Excellent", 60–89 → "Pass", else → "Fail". Can you still use a ternary? (Nested ternary allowed but discouraged – try anyway.)

- [ ] **Task 4: Simple calculator**  
  Ask the user for two numbers and an operator (`+`, `-`, `*`, `/`). Use `if-elif-else` to perform the operation. Handle division by zero gracefully.

- [ ] **Task 5: Vowel/consonant detector**  
  Input a single character. Print `"Vowel"` if it’s `a, e, i, o, u` (case‑insensitive), `"Consonant"` if it’s a letter, otherwise `"Not a letter"`.

## 🔘 Nested If

- [ ] **Task 1: Admission eligibility**  
  Write a program that asks for a student’s marks in Maths, Physics, and Chemistry.  
  - If total marks ≥ 180 → eligible.  
    - If Maths ≥ 60 and Physics ≥ 60 → print `"Eligible for Engineering"`.  
    - Else if Biology ≥ 70 → print `"Eligible for Medical"`.  
    - Else → `"Eligible for General Science"`.  
  - Else → `"Not eligible"`.  
  *Use nested `if` inside the `total >= 180` block.*

- [ ] **Task 2: Login system with role check**  
  Simulate a login system:
  1. Ask for username and password.  
  2. If username == `"admin"` and password == `"admin123"`:  
     - Inside, ask `"Enter role (user/superuser):"`  
       - If `superuser` → print `"Full system access"`  
       - Else → print `"Limited access"`  
  3. Else if username == `"guest"` and password == `"guest"`:  
     - Print `"Read‑only access"`  
  4. Else → `"Invalid credentials"`.

- [ ] **Task 3: Shipping cost calculator**  
  Calculate shipping cost based on:
  - If order value > 100 → free shipping.
    - If express delivery requested → additional $10.
    - Else → free.
  - Else (order ≤ 100):
    - If weight < 2 kg → $5.
    - If weight 2–5 kg → $10.
    - If weight > 5 kg → $20.
    - Plus, if express → add $5.
  *Use nested `if` inside both the `> 100` and `≤ 100` branches.*

- [ ] **Task 4: Refactor to guard clauses**  
  Given this deeply nested code, rewrite it using **guard clauses** (early `return` statements) without changing logic:
  ```python
  def process(user):
      if user.active:
          if user.verified:
              if user.has_permission:
                  return "Access granted"
              else:
                  return "No permission"
          else:
              return "Not verified"
      else:
          return "Account inactive"
  ```

- [ ] **Task 5: Triangle type classifier**  
  Ask for three side lengths. First check if they form a valid triangle (sum of any two > third). If valid, nest inside to classify as:
  - Equilateral (all sides equal)
  - Isosceles (exactly two sides equal)
  - Scalene (all different)

## 🔘 Match Case

- [ ] **Task 1: Command dispatcher**  
  Write a program that repeatedly asks the user for a command (string like `"hello"`, `"quit"`, `"add 3 5"`, `"subtract 10 4"`).  
  Use `match-case` to:
  - `"hello"` → print `"Hi there!"`
  - `"quit"` → break the loop
  - `"add x y"` → print sum of integers x and y
  - `"subtract x y"` → print difference x - y
  - `_` → `"Unknown command"`

- [ ] **Task 2: HTTP status code handler**  
  Write a function that takes an HTTP status code and returns a message using `match`:
  - 200 → `"OK"`
  - 201 → `"Created"`
  - 400, 401, 403, 404 → `"Client error"`
  - 500, 502, 503 → `"Server error"`
  - `_` → `"Unknown status"`
  Use **OR patterns** (`|`) for the error groups.

- [ ] **Task 3: Destructuring coordinates**  
  Given a tuple `point = (x, y)`, use `match` to print:
  - `"Origin"` for (0,0)
  - `"On X axis"` if y == 0
  - `"On Y axis"` if x == 0
  - `"Quadrant I"` if x > 0 and y > 0
  - `"Quadrant II"` if x < 0 and y > 0
  - etc. (all four quadrants)
  *Use guards (`if`) inside patterns.*

- [ ] **Task 4: Parsing JSON‑like data**  
  Write a function that processes a configuration dictionary:
  ```python
  config = {"type": "database", "host": "localhost", "port": 5432}
  ```
  Use `match` to handle:
  - `{"type": "database", "host": h, "port": p}` → print `"DB at {h}:{p}"`
  - `{"type": "api", "endpoint": e, "method": m}` → print `"API {m} {e}"`
  - `{"type": "cache", "ttl": t}` → print `"Cache TTL {t}s"`
  - `_` → `"Unknown config type"`

- [ ] **Task 5: Wildcard and capture**  
  Explain the difference between `case _:` and `case other:` in a `match` statement. Write a small example where `case other:` would cause unexpected behaviour if you meant to ignore the value. Then fix it.

## 🔘 Logical Problems

- [ ] **Task 1: Truth table generator**  
  Write a program that prints the truth table for `(A and B) or (not C)` for all combinations of A, B, C in {True, False}. Use nested loops or `itertools.product`. Format the output as a table.

- [ ] **Task 2: Simplify using De Morgan**  
  Simplify the expression `not (x > 5 and y < 10)` using De Morgan’s law. Then write a Python condition that is logically equivalent but uses no `not` parentheses around the `and`. Verify with a few test cases.

- [ ] **Task 3: Safe division with short‑circuit**  
  Write a one‑liner that divides `a` by `b` only if `b != 0`, otherwise returns `None`. Use the `and` short‑circuit trick. Then write the same using a ternary.

- [ ] **Task 4: XOR without `!=`**  
  Implement a function `xor(a, b)` that returns `True` if exactly one of `a` or `b` is `True`. You may only use `and`, `or`, and `not` (no `!=`, no `^`). Test with all four combinations.

- [ ] **Task 5: Knights and Knaves puzzle**  
  On an island, knights always tell the truth, knaves always lie. You meet two people, A and B.  
  A says: “B is a knight.”  
  B says: “A and I are of opposite types.”  
  Determine which type each is. Solve by brute force (try all 4 possibilities) and using logical expressions. Print the result.

- [ ] **Task 6: Filtering with `any()` and `all()`**  
  Given a list of numbers `[12, 45, 67, 23, 89, 34]`, write conditions that check:
  - Are all numbers > 10?
  - Is there any number > 100?
  - Are at least three numbers even?
  Use `all()` and `any()` with generator expressions.

- [ ] **Task 7: Rule‑based eligibility**  
  Write a function `can_borrow(age, has_card, late_fees, income)` that returns `True` only if:
  - Age ≥ 18 **AND**
  - Has library card **OR** (income > 30000 **AND** no late fees)
  - **AND** (late fees < 50 **OR** age > 65)  
  Test it with at least 5 different input combinations and print the result.

## 📌 How to Approach

1. Try each task **without looking up solutions** first.
2. Run your code with different test values.
3. If stuck, break the problem into smaller pieces (e.g., write the conditions separately).
4. After solving, compare with a peer or look for a more Pythonic way.
5. For bonus, add input validation and error handling.

Happy coding! 🐍
