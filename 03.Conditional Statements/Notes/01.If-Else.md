**If-Else**

## 1. Introduction

If-else statements enable conditional logic and decision-making in code. Python provides `if`, `elif`, and `else` to control execution flow based on conditions. They exist because real-world ML systems must handle different data scenarios, edge cases, and configurations dynamically. They are fundamental for building robust preprocessing, model selection, validation, and production inference logic.

## 2. Core Theory

- **Definition**: Conditional branching that executes different code blocks based on boolean expressions.
- **Syntax**:
  ```python
  if condition:
      # code
  elif another_condition:
      # code
  else:
      # code
  ```
- **Key Properties**:
  - Conditions are evaluated sequentially.
  - Uses short-circuit evaluation.
  - Supports `and`, `or`, `not`, and any truthy/falsy values.
  - No switch-case (use `if-elif` or `match` in Python 3.10+).

## 3. Core Methods / Operations

**Basic If-Else**
```python
def check_threshold(score: float) -> str:
    if score >= 0.9:
        return "Excellent"
    elif score >= 0.7:
        return "Good"
    else:
        return "Needs Improvement"
```

**Ternary Operator** (concise if-else)
```python
status = "High" if accuracy > 0.85 else "Low"
```

**Logical Operators**
```python
if epoch > 50 and val_loss < 0.1 or early_stop:
    ...
```

**Truthiness**
- Falsy: `False`, `0`, `""`, `[]`, `{}`, `None`
- Truthy: everything else.

1. **Backend APIs**: Input validation and error handling.
2. **Automation Scripts**: Environment-specific configuration loading.
3. **Data Processing**: Conditional feature engineering and cleaning.
4. **Machine Learning**: Model selection, dynamic learning rate scheduling, and inference routing.
5. **Production Software**: A/B testing logic and fallback mechanisms.

- **Scikit-learn**: Conditional pipeline steps and hyperparameter selection.
- **PyTorch/TensorFlow**: Dynamic model behavior, gradient clipping conditions, early stopping.
- **Pandas**: Used inside `apply()` or custom functions for row-wise logic.
- **Experiment Tracking**: Different training regimes based on dataset size or hardware.

## 6. Common Mistakes

1. **Deeply nested if-else**  
   Wrong: Multiple levels of nesting.  
   Correct: Early returns or guard clauses.  
   Why: Hard to read and maintain.

2. **Using assignment in condition**  
   Wrong: `if x = 5:`  
   Correct: `if (x := 5):` (walrus) or separate assignment.

3. **Redundant boolean checks**  
   Wrong: `if flag == True:`  
   Correct: `if flag:`

4. **Floating point direct equality**  
   Wrong: `if loss == 0.0:`  
   Correct: `if abs(loss) < 1e-8:`

5. **Ignoring else in exhaustive checks**  
   Wrong: Missing default case.  
   Correct: Always include `else` or raise error for unexpected states.

## 7. Practical Examples

**1. Basic Example**
```python
def is_valid_split(ratio: float) -> bool:
    if 0 < ratio < 1:
        return True
    return False
```

**2. Intermediate Example**
```python
def adjust_lr(epoch: int, lr: float) -> float:
    if epoch < 10:
        return lr
    elif epoch < 50:
        return lr * 0.1
    else:
        return lr * 0.01
```

**3. Logic Building Example**
```python
def categorize_sample(features: dict) -> str:
    if features.get("age", 0) > 65 and features.get("income", 0) > 100000:
        return "High Value Senior"
    elif features.get("income", 0) > 50000:
        return "Standard"
    else:
        return "Low Priority"
```

**4. Real Software Example**
```python
def load_config(env: str) -> dict:
    if env == "production":
        return {"debug": False, "batch_size": 512}
    elif env == "staging":
        return {"debug": True, "batch_size": 64}
    else:  # development
        return {"debug": True, "batch_size": 32}
```

**5. Production / ML Example**
```python
import torch

def training_step(model, batch, epoch: int, phase: str):
    if phase == "train":
        model.train()
        optimizer = torch.optim.Adam(model.parameters(), lr=0.001)
    elif phase == "val":
        model.eval()
        optimizer = None
    else:
        raise ValueError(f"Unknown phase: {phase}")
    
    # ... training logic with early stopping
    if epoch > 100 and val_loss < best_loss:
        best_loss = val_loss
        torch.save(model.state_dict(), "best_model.pth")
```

## 8. Time Complexity

Conditional statements themselves have negligible cost (O(1) per evaluation). Complexity depends on the expressions inside conditions.

| Operation          | Time Complexity |
|--------------------|-----------------|
| Simple condition   | O(1)            |
| Multiple `elif`    | O(k) worst case (k branches) |
| Short-circuit `and`/`or` | Early exit benefit |

## 9. Key Takeaways

- Use clear, flat conditional logic with early returns.
- Prefer ternary for simple assignments.
- Always handle the `else` case or raise explicit errors.
- Leverage truthiness but be explicit for clarity in critical paths.
- Combine with `match` (Python 3.10+) for complex branching.
- Guard clauses improve readability in ML preprocessing functions.
- Use conditions for dynamic behavior in training loops and inference.
- Test edge cases thoroughly in production ML systems.
- Keep conditions simple — complex logic belongs in separate functions.
- Master if-else patterns for robust, maintainable ML pipelines.
