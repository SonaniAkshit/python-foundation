**Nested Dictionaries**

## 1. Introduction

Nested dictionaries are dictionaries that contain other dictionaries (or lists) as values. They exist to represent hierarchical, structured, and semi-structured data naturally. They are crucial in ML engineering for handling complex configurations, JSON-like data, model architectures, feature hierarchies, and experiment tracking metadata.

## 2. Core Theory

- **Definition**: A dictionary where values can themselves be dictionaries, enabling multi-level key access (e.g., `d["level1"]["level2"]`).
- **Syntax**: `{"outer": {"inner": value}}` or built dynamically.
- **Key Properties**:
  - Mutable at every level.
  - Keys remain hashable at all levels.
  - No built-in limit on depth (but deep nesting hurts readability and performance).
  - Common in JSON deserialization.

## 3. Core Methods / Operations

**Accessing & Safe Retrieval**
```python
config: dict = {"training": {"lr": 0.001, "epochs": 100}}
print(config["training"]["lr"])                    # direct access
print(config.get("training", {}).get("lr", 0.01))  # safe nested get
```

**`setdefault()` for nested creation**
```python
data: dict = {}
data.setdefault("model", {}).setdefault("optimizer", {})["name"] = "Adam"
```

**Updating nested structures**
```python
def deep_update(d: dict, u: dict) -> dict:
    for k, v in u.items():
        if isinstance(v, dict) and k in d and isinstance(d[k], dict):
            deep_update(d[k], v)
        else:
            d[k] = v
    return d
```

**Flattening & traversal** (common in production)
- Manual recursion or libraries like `flatten_dict`.

## 4. Production Use Cases

1. **Backend APIs**: Parsing complex JSON payloads from external services.
2. **Automation Scripts**: Loading multi-level YAML/JSON configuration files.
3. **Data Processing**: Representing hierarchical features or nested records.
4. **Machine Learning**: Storing model architectures, layer configurations, or experiment results.
5. **Production Software**: Feature store metadata, logging nested metrics, and MLflow-style run tracking.

## 5. Machine Learning Connection

- **Pandas**: `df.to_dict(orient="records")` often produces nested structures; `json_normalize` for flattening.
- **TensorFlow/PyTorch**: Model config dicts, `state_dict` with nested modules, optimizer parameter groups.
- **Scikit-learn**: Pipeline parameter grids with nested steps.
- **Hugging Face / ML frameworks**: `model.config` objects and training arguments as nested dicts.
- **Experiment tracking**: Storing hyperparameters, metrics history, and artifacts.

## 6. Common Mistakes

1. **Direct access without existence check**  
   Wrong: `val = d["model"]["lr"]`  
   Correct: `val = d.get("model", {}).get("lr")`  
   Why: KeyError on missing levels.

2. **Shallow copy on nested structures**  
   Wrong: `new_d = old_d.copy()` then modify inner dict.  
   Correct: Use `copy.deepcopy(old_d)` or manual deep copy.

3. **Excessive nesting depth**  
   Wrong: 5+ levels of nesting.  
   Correct: Flatten or use dataclasses/Pydantic models.

4. **Mutation during traversal**  
   Wrong: Modifying nested dict while iterating.  
   Correct: Collect changes separately or use list of keys.

5. **Inefficient repeated nested access**  
   Wrong: Repeated `config["a"]["b"]["c"]` in loops.  
   Correct: Cache reference: `inner = config["a"]["b"]`.

## 7. Practical Examples

**1. Basic Example**
```python
hyperparams: dict[str, dict] = {
    "optimizer": {"name": "Adam", "lr": 0.001},
    "scheduler": {"type": "cosine", "T_max": 100}
}
print(hyperparams.get("optimizer", {}).get("lr"))
```
Output: `0.001`

**2. Intermediate Example**
```python
def safe_nested_get(d: dict, keys: list[str], default=None):
    for k in keys:
        d = d.get(k, {}) if isinstance(d, dict) else {}
    return d if d != {} else default

print(safe_nested_get({"model": {"layers": 4}}, ["model", "layers"]))
```
Output: `4`

**3. Logic Building Example**
```python
def build_feature_hierarchy(features: list[str]) -> dict:
    hierarchy: dict = {}
    for f in features:
        parts = f.split("_")
        current = hierarchy
        for part in parts[:-1]:
            current = current.setdefault(part, {})
        current[parts[-1]] = True
    return hierarchy

print(build_feature_hierarchy(["user_age", "user_income", "item_price"]))
```
Output: `{'user': {'age': True, 'income': True}, 'item': {'price': True}}`

**4. Real Software Example**
```python
from copy import deepcopy

def merge_experiment_configs(base: dict, override: dict) -> dict:
    merged = deepcopy(base)
    deep_update(merged, override)  # using deep_update from earlier
    return merged

base = {"training": {"lr": 0.01, "epochs": 50}}
override = {"training": {"lr": 0.001}}
print(merge_experiment_configs(base, override))
```

**5. Production / ML Example**
```python
import torch.nn as nn

def get_model_config(model: nn.Module) -> dict:
    config: dict = {"architecture": {}, "parameters": {}}
    for name, module in model.named_modules():
        if len(list(module.children())) == 0:  # leaf
            config["architecture"][name] = module.__class__.__name__
            config["parameters"][name] = sum(p.numel() for p in module.parameters())
    return config

# Usage
model = nn.Sequential(nn.Linear(10, 5), nn.ReLU(), nn.Linear(5, 1))
config = get_model_config(model)
print(config["parameters"])  # layer-wise param counts
```

## 8. Time Complexity

| Operation                  | Time Complexity      |
|----------------------------|----------------------|
| Nested access `d[k1][k2]`  | O(1) average per level |
| Deep update (recursive)    | O(N) where N = total keys |
| Deep copy                  | O(N)                 |
| Flattening                 | O(N)                 |
| Iteration over all leaves  | O(N)                 |

## 9. Key Takeaways

- Always use safe access patterns (`get().get()`) or helper functions for nested dicts.
- Prefer `setdefault()` for building nested structures incrementally.
- Implement or use `deep_update` and `deepcopy` when merging or copying.
- Keep nesting shallow; consider dataclasses, Pydantic, or namedtuples for complex cases.
- Nested dicts are the natural representation for JSON/YAML configs in ML.
- Master flattening (`pandas.json_normalize`) and traversal for data pipelines.
- Cache inner references in performance-critical loops.
- In PyTorch/TF, understand how `state_dict()` creates nested structures.
- Combine with `defaultdict` for cleaner hierarchical grouping.
- Validate nested configs early in production ML systems.
