**String vs List vs Tuple vs Set vs Dictionary**

## 1. Introduction

Python provides several built-in data structures — strings, lists, tuples, sets, and dictionaries — each optimized for specific access patterns and use cases. They differ in mutability, ordering, uniqueness, and performance. Choosing the right one is critical for efficient data handling, memory usage, and performance in ML pipelines, feature engineering, and production systems.

## 2. Core Theory

- **String (`str`)**: Immutable sequence of Unicode characters. Used for text and fixed data.
- **List (`list`)**: Mutable, ordered, allows duplicates. Dynamic array.
- **Tuple (`tuple`)**: Immutable, ordered, allows duplicates. Fixed-size.
- **Set (`set`)**: Mutable, unordered, unique elements only. Hash table.
- **Dictionary (`dict`)**: Mutable, insertion-ordered (3.7+), unique keys mapping to values. Hash table.

**Key Differences**:
- Mutability: String/Tuple immutable; List/Set/Dict mutable.
- Ordering: String/List/Tuple/Dict (since 3.7) ordered; Set unordered.
- Uniqueness: Set (elements), Dict (keys).
- Access: Sequence types by index; Dict/Set by hash.

## 3. Core Methods / Operations

**Common Operations**
- **Indexing/Slicing**: String/List/Tuple (`[i]`, `[i:j]`).
- **Membership (`in`)**: All support, fastest in Set/Dict (O(1) avg).
- **Length**: `len()` for all (O(1)).

**Type-Specific**
- List: `append()`, `extend()`, `pop()`, `sort()`.
- Tuple: No mutators; good for unpacking.
- Set: `add()`, `remove()`, `union()`, `intersection()`.
- Dict: `get()`, `update()`, `keys()/values()/items()`.
- String: `split()`, `join()`, `strip()`, `format()`.

**Example Comparison**
```python
s = "hello"          # immutable
lst = [1, 2, 2]      # mutable, ordered, duplicates
tpl = (1, 2, 2)      # immutable
st = {1, 2}          # unique, unordered
d = {"a": 1, "b": 2} # key-value
```

## 4. Production Use Cases

1. **Backend APIs**: Strings for text/JSON keys, Dicts for payloads, Lists for arrays.
2. **Automation Scripts**: Tuples for fixed coordinates/configs, Sets for deduplication.
3. **Data Processing**: Lists for sequences, Sets for unique IDs, Dicts for mappings.
4. **Machine Learning**: Dicts for hyperparameters/configs, Lists for batches, Tuples for dataset splits.
5. **Production Software**: Sets for fast lookup of valid categories, Tuples as dict keys or return values.

## 5. Machine Learning Connection

- **NumPy/Pandas**: Lists for column selection, Dicts for `rename()` or `to_dict()`, Tuples for multi-index.
- **Scikit-learn**: Lists for feature lists, Dicts for `param_grid`, Sets for unique classes.
- **TensorFlow/PyTorch**: Tuples for shape (`(batch, channels, h, w)`), Dicts for `state_dict()`, Lists for DataLoader batches.
- **Data Loading**: Strings for file paths, Sets for vocabulary, Dicts for token-to-id mappings.

## 6. Common Mistakes

1. **Trying to modify immutable types**  
   Wrong: `s[0] = "H"` or `t[0] = 99`  
   Correct: `s = "H" + s[1:]` or create new tuple.  
   Why: TypeError.

2. **Using list when set is better for lookups**  
   Wrong: `if x in large_list` in loop.  
   Correct: Convert to set for O(1) checks.

3. **Assuming dict/set order (pre-3.7 or across versions)**  
   Wrong: Relying on iteration order for Sets.  
   Correct: Use List or `sorted()` when order matters.

4. **Storing unhashable types in sets/dict keys**  
   Wrong: `{(1,2): "ok", [3,4]: "bad"}`  
   Correct: Use tuples for compound keys.

5. **Unnecessary copying of large lists/tuples**  
   Wrong: Passing large lists without care in recursive functions.  
   Correct: Use tuples or references wisely.

## 7. Practical Examples

**1. Basic Example**
```python
text: str = "machine learning"
words: list[str] = text.split()
unique: set[str] = set(words)
print(len(words), len(unique))
```
Output: `2 2`

**2. Intermediate Example**
```python
features: tuple[str, ...] = ("age", "income", "age")  # immutable
feature_set: set[str] = set(features)
print("age" in feature_set)  # True, fast
```

**3. Logic Building Example**
```python
def dedup_and_map(data: list[str]) -> tuple[list[str], dict[str, int]]:
    unique = list(dict.fromkeys(data))  # preserve order
    mapping = {val: idx for idx, val in enumerate(unique)}
    return unique, mapping

print(dedup_and_map(["cat", "dog", "cat"]))
```
Output: `(['cat', 'dog'], {'cat': 0, 'dog': 1})`

**4. Real Software Example**
```python
def validate_batch(batch: list[dict]) -> bool:
    required: tuple[str, ...] = ("id", "features")
    for sample in batch:
        if not all(k in sample for k in required):
            return False
    return True
```

**5. Production / ML Example**
```python
from torch.utils.data import DataLoader

# Config as dict, shapes as tuple, vocab as set
config: dict = {"batch_size": 32, "lr": 0.001}
input_shape: tuple[int, ...] = (batch_size := 32, 3, 224, 224)
vocab: set[str] = {"token1", "token2"}

def collate_fn(batch):
    return {
        "inputs": [item["features"] for item in batch],  # list
        "labels": tuple(item["label"] for item in batch)  # tuple
    }
```

## 8. Time Complexity

| Operation              | String | List     | Tuple | Set    | Dict     |
|------------------------|--------|----------|-------|--------|----------|
| Access by index/key    | O(1)   | O(1)     | O(1)  | -      | O(1) avg |
| Membership (`in`)      | O(n)   | O(n)     | O(n)  | O(1)   | O(1) avg |
| Add/Insert             | O(n)   | O(1) amm | -     | O(1)   | O(1)     |
| Delete                 | O(n)   | O(n)     | -     | O(1)   | O(1)     |
| Iteration              | O(n)   | O(n)     | O(n)  | O(n)   | O(n)     |

## 9. Key Takeaways

- Choose **String** for text manipulation and paths.
- Use **List** for ordered, mutable sequences and dynamic collections.
- Prefer **Tuple** for fixed data, function returns, and hashable keys.
- Use **Set** for uniqueness and blazing-fast membership tests.
- Use **Dict** for key-value mappings, configurations, and metadata.
- In ML, combine them: dict of lists, tuple of shapes, set of features.
- Convert between types liberally (`list(set())`, `dict.fromkeys()`).
- Favor immutability (tuples) for safer, hashable data in production.
- Profile carefully: Sets/Dicts win on lookups; Lists win on order/sequences.
- Master these differences to write cleaner, faster ML data pipelines.
