# Indexing & Slicing in Python

## 1. Intuitive Introduction

Strings, lists, tuples, and many Python data structures store data in sequence form.

Example:

```python
name = "Akshit"
```

Internally:

| Position | Character |
| -------- | --------- |
| 0        | A         |
| 1        | k         |
| 2        | s         |
| 3        | h         |
| 4        | i         |
| 5        | t         |

Sometimes you need:

* one specific item
* a range of items
* reverse data
* skip items
* extract patterns

That is exactly why **indexing** and **slicing** exist.

Without them:

* data extraction becomes painful
* ML preprocessing becomes harder
* text processing becomes slow
* dataset manipulation becomes messy

This is one of the MOST used concepts in:

* Python
* Data Science
* Machine Learning
* Backend Engineering
* APIs
* NLP
* Pandas
* NumPy

# 3. Core Theory

---

# What is Indexing?

Indexing means:

> accessing a single element using its position

Example:

```python
word = "Python"

print(word[0])
```

Output:

```python
P
```

---

# What is Slicing?

Slicing means:

> extracting a portion of a sequence

Example:

```python
word = "Python"

print(word[0:4])
```

Output:

```python
Pyth
```

# Internal Working

Python sequences internally store:

* ordered elements
* memory references
* positional access mapping

When you do:

```python
name[2]
```

Python:

1. locates sequence object
2. jumps to index position
3. returns object reference

Very fast operation.

Time Complexity:

| Operation | Complexity |
| --------- | ---------- |
| Indexing  | O(1)       |
| Slicing   | O(k)       |

Why slicing is slower:

Because Python creates a NEW object.

| Part  | Meaning        |
| ----- | -------------- |
| start | where to begin |
| end   | where to stop  |

# 6. Memory + Internal Working

# Strings are Immutable

```python
name = "Python"
```

You cannot do:

```python
name[0] = "J"
```

ERROR.

Because strings are immutable.

Python creates new objects instead.

---

# Slicing Creates New Object

```python
a = "Python"
b = a[0:3]
```

Memory behavior:

```mermaid
flowchart TD
    A["Python"] --> B["Pyt"]
```

New object created.

This matters in:

* large datasets
* NLP pipelines
* big text processing

Too much slicing = memory overhead.

---

# Lists vs Strings

Lists are mutable.

```python
nums = [1, 2, 3]

nums[0] = 100
```

Allowed.

# 11. Performance Considerations

# Indexing

```python
text[5]
```

Complexity:

```text
O(1)
```

Very fast.

---

# Slicing

```python
text[0:5000]
```

Complexity:

```text
O(k)
```

Because new object creation happens.

---

# Optimization Tip

Avoid repeated slicing inside loops.

BAD:

```python
for i in range(len(text)):
    print(text[:i])
```

Creates many new objects.

---

# Better Approach

Use indices intelligently.

# 12. Debugging Mindset

When debugging slicing:

```python

```

Print lengths:

```python

```

Trace indices carefully.

text = "Python"
