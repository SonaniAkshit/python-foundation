# The Complete Python Collections Method Sheet

This cheat sheet provides a quick-reference guide to the most essential methods for Python’s five core collection types: **Strings**, **Lists**, **Tuples**, **Sets**, and **Dictionaries**.

---

## 1. String Methods (`str`)

*Strings are immutable character sequences. All transformation methods return a brand new string.*

| Method | Syntax & Parameters | Return Type | Description |
| --- | --- | --- | --- |
| **`.strip()`** | `s.strip([chars])` | `str` | Removes leading and trailing whitespace (or specific characters). |
| **`.lower()`** | `s.lower()` | `str` | Converts all characters in the string to lowercase. |
| **`.upper()`** | `s.upper()` | `str` | Converts all characters in the string to uppercase. |
| **`.replace()`** | `s.replace(old, new[, count])` | `str` | Replaces occurrences of a substring with a new substring. |
| **`.split()`** | `s.split(sep=None, maxsplit=-1)` | `list[str]` | Splits the string into a list of substrings using a delimiter. |
| **`.join()`** | `sep.join(iterable)` | `str` | Concatenates elements of an iterable of strings, separated by `sep`. |
| **`.find()`** | `s.find(sub[, start[, end]])` | `int` | Returns the lowest index of `sub` if found; returns `-1` if missing. |
| **`.index()`** | `s.index(sub[, start[, end]])` | `int` | Like `.find()`, but raises a `ValueError` if the substring is missing. |
| **`.startswith()`** | `s.startswith(prefix[, start[, end]])` | `bool` | Returns `True` if the string starts with the specified prefix. |
| **`.endswith()`** | `s.endswith(suffix[, start[, end]])` | `bool` | Returns `True` if the string ends with the specified suffix. |
| **`.isdigit()`** | `s.isdigit()` | `bool` | Returns `True` if all characters in the string are digits. |
| **`.isalpha()`** | `s.isalpha()` | `bool` | Returns `True` if all characters in the string are alphabetic. |

---

## 2. List Methods (`list`)

*Lists are mutable, ordered arrays of object pointers. Most mutation methods modify the list **in-place** and return `None`.*

| Method | Syntax & Parameters | Return Type | Description | Time Complexity |
| --- | --- | --- | --- | --- |
| **`.append()`** | `l.append(item)` | `None` | Adds an element to the end of the list. | $O(1)$ (Amortized) |
| **`.extend()`** | `l.extend(iterable)` | `None` | Appends all elements from an iterable to the end of the list. | $O(k)$ |
| **`.insert()`** | `l.insert(index, item)` | `None` | Inserts an item at a specified index, shifting trailing items. | $O(n)$ |
| **`.remove()`** | `l.remove(item)` | `None` | Removes the first occurrence of `item`. Raises `ValueError` if missing. | $O(n)$ |
| **`.pop()`** | `l.pop([index])` | `Value` | Removes and returns the item at `index` (defaults to the last item). | $O(1)$ last / $O(n)$ index |
| **`.clear()`** | `l.clear()` | `None` | Removes all elements from the list, leaving it empty. | $O(n)$ |
| **`.index()`** | `l.index(item[, start[, end]])` | `int` | Returns the zero-based index of the first occurrence of `item`. | $O(n)$ |
| **`.count()`** | `l.count(item)` | `int` | Returns the number of times `item` appears in the list. | $O(n)$ |
| **`.sort()`** | `l.sort(key=None, reverse=False)` | `None` | Sorts the items of the list **in-place**. | $O(n \log n)$ |
| **`.reverse()`** | `l.reverse()` | `None` | Reverses the elements of the list **in-place**. | $O(n)$ |
| **`.copy()`** | `l.copy()` | `list` | Returns a shallow copy of the list. | $O(n)$ |

---

## 3. Tuple Methods (`tuple`)

*Tuples are immutable, fixed-size sequences. They only support read-only operations.*

| Method | Syntax & Parameters | Return Type | Description | Time Complexity |
| --- | --- | --- | --- | --- |
| **`.count()`** | `t.count(item)` | `int` | Returns the number of times `item` appears in the tuple. | $O(n)$ |
| **`.index()`** | `t.index(item[, start[, end]])` | `int` | Returns the zero-based index of the first occurrence of `item`. | $O(n)$ |

---

## 4. Set Methods (`set`)

*Sets are unordered collections of unique, hashable items. They support powerful mathematical operations.*

| Method | Syntax & Parameters | Return Type | Description | Time Complexity |
| --- | --- | --- | --- | --- |
| **`.add()`** | `s.add(item)` | `None` | Adds an element to the set if it isn't already present. | $O(1)$ |
| **`.remove()`** | `s.remove(item)` | `None` | Removes `item` from the set. Raises `KeyError` if missing. | $O(1)$ |
| **`.discard()`** | `s.discard(item)` | `None` | Removes `item` from the set safely; does nothing if missing. | $O(1)$ |
| **`.pop()`** | `s.pop()` | `Value` | Removes and returns an arbitrary element. Raises `KeyError` if empty. | $O(1)$ |
| **`.clear()`** | `s.clear()` | `None` | Removes all elements from the set. | $O(n)$ |
| **`.union()`** | `s1.union(s2, ...)` or `s1 | s2` | `set` | Returns a new set with elements from both sets. | $O(n + m)$ |
| **`.intersection()`** | `s1.intersection(s2)` or `s1 & s2` | `set` | Returns a new set containing only elements common to both sets. | $O(\min(n, m))$ |
| **`.difference()`** | `s1.difference(s2)` or `s1 - s2` | `set` | Returns a new set with elements in `s1` that are not in `s2`. | $O(n)$ |
| **`.symmetric_difference()`** | `s1.symmetric_difference(s2)` or `s1 ^ s2` | `set` | Returns a new set with elements in either set, but not both. | $O(n)$ |
| **`.issubset()`** | `s1.issubset(s2)` | `bool` | Returns `True` if all elements of `s1` are present in `s2`. | $O(n)$ |
| **`.issuperset()`** | `s1.issuperset(s2)` | `bool` | Returns `True` if `s1` contains every element found in `s2`. | $O(m)$ |
| **`.isdisjoint()`** | `s1.isdisjoint(s2)` | `bool` | Returns `True` if `s1` and `s2` share zero elements in common. | $O(\min(n, m))$ |

## 5. Dictionary Methods (`dict`)

*Dictionaries are mutable key-value maps. Key mutations happen **in-place**.*

| Method | Syntax & Parameters | Return Type | Description | Time Complexity |
| --- | --- | --- | --- | --- |
| **`.get()`** | `d.get(key[, default])` | `Value` | Safely fetches a value. Returns `default` (or `None`) if `key` is missing. | $O(1)$ |
| **`.setdefault()`** | `d.setdefault(key, default)` | `Value` | Returns value if `key` exists; otherwise inserts `key` with `default`. | $O(1)$ |
| **`.update()`** | `d.update(iterable)` | `None` | Merges key-value pairs from another mapping or iterable in-place. | $O(k)$ |
| **`.pop()`** | `d.pop(key[, default])` | `Value` | Removes specific `key` and returns its value. Errors if missing without a default. | $O(1)$ |
| **`.popitem()`** | `d.popitem()` | `tuple` | Removes and returns the last inserted `(key, value)` pair as a tuple. | $O(1)$ |
| **`.clear()`** | `d.clear()` | `None` | Wipes out all keys and values from the dictionary. | $O(n)$ |
| **`.keys()`** | `d.keys()` | `dict_keys` | Returns a live, dynamic view object of all keys in the dictionary. | $O(1)$ |
| **`.values()`** | `d.values()` | `dict_values` | Returns a live, dynamic view object of all values in the dictionary. | $O(1)$ |
| **`.items()`** | `d.items()` | `dict_items` | Returns a live, dynamic view object of all `(key, value)` tuples. | $O(1)$ |
| **`.copy()`** | `d.copy()` | `dict` | Returns a top-level shallow copy of the dictionary. | $O(n)$ |
