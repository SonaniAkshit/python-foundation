**Dictionary Methods**

## 1. Introduction

Python dictionaries (`dict`) are mutable, unordered (insertion-ordered since Python 3.7) collections of key-value pairs that provide average O(1) lookup time. They exist because efficient key-based access is fundamental for fast data retrieval and organization. In ML engineering, dictionaries are indispensable for configuration management, feature mapping, hyperparameter storage, and intermediate data processing pipelines.

## 2. Core Theory

- **Definition**: A mapping type where keys must be hashable (immutable) and unique; values can be any type.
- **Syntax**: `{"key": value}` or `dict(key=value)`.
- **Key Properties**:
  - Keys: hashable (str, int, tuple, frozenset, etc.).
  - Mutable: can be modified in place.
  - Insertion order preserved (Python 3.7+).
  - Dynamic size.

## 3. Core Methods / Operations

**`get(key, default=None)`**  
Purpose: Safe key access without raising KeyError.  
Syntax: `d.get(key, default)`  
Returns: Value or default.  
```python
config: dict[str, int] = {"epochs": 100}
print(config.get("epochs"))          # 100
print(config.get("batch_size", 32))  # 32
```

**`keys()`, `values()`, `items()`**  
Purpose: View keys/values/pairs (memory-efficient views).  
```python
data = {"feature1": 0.8, "feature2": 0.3}
print(list(data.keys()))    # ['feature1', 'feature2']
print(list(data.values()))  # [0.8, 0.3]
print(list(data.items()))   # [('feature1', 0.8), ...]
```

**`update([other])`**  
Purpose: Merge another dict or iterable of pairs.  
```python
params = {"lr": 0.001}
params.update({"batch_size": 64, "lr": 0.01})  # overwrites
```

**`pop(key, default)`**  
Purpose: Remove and return value (with optional default).  
```python
metrics = {"loss": 0.45, "acc": 0.92}
loss = metrics.pop("loss")  # 0.45
```

**`setdefault(key, default)`**  
Purpose: Insert default if key missing, then return value.  
**`popitem()`**: Remove and return last inserted pair (LIFO).  
**`clear()`** and **`copy()`**: Self-explanatory.

## 4. Production Use Cases

1. **Backend APIs**: Fast request/response payload transformation and validation.
2. **Automation Scripts**: Configuration loading with environment overrides.
3. **Data Processing**: Mapping categorical features to encodings or aggregations.
4. **Machine Learning**: Storing model hyperparameters, experiment tracking metadata.
5. **Production Software**: Caching intermediate results in ETL pipelines or feature stores.

## 5. Machine Learning Connection

- **NumPy/Pandas**: `df.to_dict()`, `dict` of Series, `orient='index'` conversions.
- **Scikit-learn**: `param_grid` in GridSearchCV, `class_weight` dicts.
- **TensorFlow/PyTorch**: Model state dicts (`model.state_dict()`), optimizer configs, dataset metadata.
- **ML Pipelines**: Feature name → transformer mappings, experiment tracking (MLflow), hyperparameter optimization storage.

## 6. Common Mistakes

1. **Modifying during iteration**  
   Wrong: `for k in d: if cond: del d[k]`  
   Correct: `for k in list(d): ...` or collect keys to remove.  
   Why: RuntimeError. Best Practice: Use list(d.keys()).

2. **Using mutable as key**  
   Wrong: `d[[1,2]] = 3`  
   Correct: Use tuple `(1,2)` or frozenset.  
   Why: Lists are unhashable.

3. **Assuming order before Python 3.7**  
   Wrong: Relying on key order in older code.  
   Correct: Use `collections.OrderedDict` if strict order needed (rare).

4. **Direct KeyError access instead of get/setdefault**  
   Wrong: `val = d["missing"]`  
   Correct: `val = d.get("missing", default)`.

5. **Deep copy oversight with nested dicts**  
   Wrong: `new_d = old_d.copy()` for nested structures.  
   Correct: `import copy; copy.deepcopy()`.

## 7. Practical Examples

**1. Basic Example**
```python
hyperparams: dict[str, float | int] = {"lr": 0.01, "epochs": 50}
print(hyperparams.get("dropout", 0.0))
```
Output: `0.0`

**2. Intermediate Example**
```python
features = {"age": 25, "income": 75000}
encoded = {k: v if isinstance(v, (int, float)) else 0 for k, v in features.items()}
print(encoded)
```
Output: `{'age': 25, 'income': 75000}`

**3. Logic Building Example**
```python
def merge_configs(default: dict, override: dict) -> dict:
    config = default.copy()
    config.update(override)
    return {k: v for k, v in config.items() if v is not None}

print(merge_configs({"lr": 0.001, "batch": 32}, {"lr": 0.01}))
```
Output: `{'lr': 0.01, 'batch': 32}`

**4. Real Software Example**
```python
def process_batch(batch: list[dict]) -> dict[str, list]:
    result: dict[str, list] = {}
    for item in batch:
        for k, v in item.items():
            result.setdefault(k, []).append(v)
    return result

data = [{"feat": 1}, {"feat": 2}, {"other": 10}]
print(process_batch(data))
```
Output: `{'feat': [1, 2], 'other': [10]}`

**5. Production / ML Example**
```python
import torch

def save_model_metadata(model: torch.nn.Module, metrics: dict, path: str = "metadata.json"):
    state = {
        "model_state": {k: v.shape for k, v in model.state_dict().items()},
        "metrics": metrics,
        "config": {"lr": 1e-4, "optimizer": "Adam"}
    }
    # In production: json.dump or MLflow logging
    return state

# Usage in training loop
print(save_model_metadata(torch.nn.Linear(10, 1), {"val_loss": 0.23}))
```

## 8. Time Complexity

| Operation          | Time Complexity |
|--------------------|-----------------|
| Access `d[key]`    | O(1) average    |
| `get()`, `set()`   | O(1) average    |
| `update()`         | O(k) where k = items added |
| `pop()`            | O(1) average    |
| `len(d)`           | O(1)            |
| Iteration          | O(n)            |

## 9. Key Takeaways

- Use `get()` and `setdefault()` for safe, clean code.
- Prefer `items()` for iteration over key-value pairs.
- `update()` is the idiomatic way to merge configurations.
- Dicts are the backbone of hyperparameter and metadata handling in ML.
- Always copy before modifying if you need to preserve original data.
- Leverage dict comprehensions for concise transformations.
- Insertion order is reliable since Python 3.7 — useful for feature ordering.
- Combine with `defaultdict` from collections for cleaner grouping logic.
- Master `state_dict` patterns for PyTorch model management.
- Profile dict-heavy code in large-scale data pipelines.
