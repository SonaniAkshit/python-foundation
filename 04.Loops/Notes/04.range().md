# **range()**

## 1. Introduction

`range()` is a built-in function that returns an immutable sequence of numbers, commonly used to generate indices or control the number of iterations in loops.

Python provides it to offer a memory-efficient way to work with arithmetic sequences without creating full lists in memory (unlike Python 2's `xrange`). It is fundamental for numeric iteration and integrates deeply with the iterator protocol.

It solves the problem of cleanly generating sequences of integers for looping, slicing, and indexing without wasting memory or writing manual counters.

---

## 2. Intuition

Think of `range()` as a smart, lazy recipe book for numbers. Instead of writing down every number on a long list (which takes space), it gives you instructions: “start here, stop before this number, count by this step.” You only get the next number when you ask for it.

---

## 3. Syntax

```python
range(stop)                    # 0 to stop-1
range(start, stop)             # start to stop-1
range(start, stop, step)       # with custom step (can be negative)
```

**Examples**:
```python
range(5)          # 0,1,2,3,4
range(2, 10)      # 2,3,...,9
range(0, 10, 2)   # 0,2,4,6,8
range(10, 0, -1)  # 10,9,...,1
```

**Python 3.13+ note**: `range` objects support full sequence protocol (`__getitem__`, `__len__`, etc.) and are very efficient.

You can use it directly in `for` loops, with `list()`, `enumerate()`, `zip()`, etc.

---

## 4. Internal Working

`range` is **lazy** — it stores only `start`, `stop`, and `step` (three integers), not the entire sequence. 

When iterated, it generates values on demand using simple arithmetic. This makes it extremely memory efficient even for `range(10**9)`.

It implements the iterator protocol and the sequence protocol, so it supports indexing (`range(10)[5] == 5`) and length checks in O(1) time.

---

## 5. Built-in Methods / Operations

`range` objects support:

- `len(r)` — O(1)
- `r[i]` — O(1) indexing
- `r.index(value)` — finds position
- `r.count(value)` — usually 0 or 1
- Membership test `in` — O(1) arithmetic check
- `list(r)`, `tuple(r)` — materialization
- Negative indices and slicing (Python 3.2+)

**Practical examples**:
```python
r = range(10, 100, 10)
print(len(r))        # 9
print(r[3])          # 40
print(50 in r)       # True
```

---

## 6. Common Mistakes

- Using `range(len(lst))` instead of `enumerate(lst)`.
- Forgetting that `range(stop)` **excludes** the stop value.
- Assuming `range` creates a list (common Python 2 hangover).
- Using large `step` or negative steps incorrectly.
- Trying to modify a `range` object (immutable).

**Wrong**:
```python
for i in range(len(items)):
    print(items[i])   # Better: for item in items: or for i, item in enumerate(items):
```

---

## 7. Best Practices

- Prefer `for i in range(...)` only when you truly need the index.
- Use `enumerate()` when you need both index and value.
- Combine with `zip()` for parallel iteration.
- Use `range` for generating IDs, pagination offsets, or batch indices.
- For very large ranges, keep it lazy — avoid unnecessary `list(range(...))`.

---

## 8. Production Usage

- **Backend**: Pagination (`offset = page * PAGE_SIZE`), batch processing.
- **Data Engineering**: Splitting large datasets into chunks: `for i in range(0, len(data), batch_size)`.
- **Automation**: Retry loops with delay: `for attempt in range(MAX_RETRIES)`.
- **APIs**: Generating test data or sequence numbers.

Example:
```python
def process_in_batches(records, batch_size=100):
    for i in range(0, len(records), batch_size):
        batch = records[i:i+batch_size]
        process_batch(batch)
```

---

## 9. Machine Learning Usage

- **Training loops**: `for epoch in range(num_epochs):`
- **Batch processing**: `for i in range(0, len(dataset), batch_size)`
- **Hyperparameter tuning**: `for lr in range(10, 100, 10):` or grid search.
- **NumPy / PyTorch**: Often combined with `torch.arange()`, `np.arange()` (similar interface).
- **Data augmentation / sampling**: Generating indices for shuffling or cross-validation folds.

```python
for epoch in range(epochs):
    for i in range(0, len(train_data), batch_size):
        batch = train_data[i:i+batch_size]
        ...
```

## 11. Complexity Analysis

- **Time**: O(1) to create the `range` object. Iteration cost is O(k) where k is the number of elements yielded.
- **Space**: O(1) — constant memory regardless of range size.
