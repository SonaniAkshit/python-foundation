# **continue**

## 1. Introduction

The `continue` statement is a control flow keyword that **skips the rest of the current iteration** of the nearest enclosing loop and proceeds to the next iteration.

Python provides it to cleanly skip unwanted items during iteration without deep nesting or complex `if-else` structures. It complements `break` — while `break` exits completely, `continue` jumps to the next cycle.

It solves the problem of filtering items inside a loop without duplicating code or creating extra boolean flags.

---

## 2. Intuition

Imagine a recruiter scanning resumes: for each resume, if it’s missing the required experience section, they **skip** it and move to the next one without further review. `continue` is exactly that “skip this one, go to the next” action.

---

## 3. Syntax

```python
for item in iterable:
    if skip_condition(item):
        continue          # jump to next iteration
    # code that runs only for kept items
    process(item)

while condition:
    ...
    if should_skip:
        continue
    ...
```

`continue` works with both `for` and `while` loops, and in nested loops it affects only the innermost loop.

---

## 4. Internal Working

When `continue` is executed:
- Python skips the remaining statements in the current iteration.
- For `for` loops: it advances the iterator and fetches the next item.
- For `while` loops: it jumps back to re-evaluate the loop condition.
- The `else` clause of the loop is **not** affected (it still runs if no `break` occurred).

It does not create new scope or extra overhead — it’s a simple jump in the control flow.

---

## 5. Built-in Methods / Operations

`continue` is a statement. Common patterns include:

- Filtering inside loops
- Combining with `enumerate()`, `range()`, `zip()`
- Using with `try/except` to skip bad items

**Practical example**:
```python
for resume in resumes:
    if not is_valid_pdf(resume):
        continue                    # skip corrupted files
    if "Python" not in resume.skills:
        continue                    # skip irrelevant resumes
    process_resume(resume)
```

---

## 6. Common Mistakes

- Placing code after `continue` that should always run (it gets skipped).
- Overusing `continue` making logic hard to follow (deeply nested conditions).
- Using `continue` in the wrong loop level in nested setups.
- Forgetting that `continue` in `while` re-checks the condition (can cause infinite loops if state not updated).
- Using `continue` instead of list comprehensions or `filter()` for simple cases.

---

## 7. Best Practices

- Use `continue` to keep main loop logic clean and focused on the “happy path”.
- Prefer list comprehensions or generator expressions for simple filtering.
- Keep conditions for `continue` near the top of the loop.
- Combine with `break` and `else` for powerful loop control.
- Document why an item is being skipped when the reason is non-obvious.

---

## 8. Production Usage

- **Backend**: Skipping invalid records in data processing pipelines, rate-limited API calls.
- **Data Engineering / ETL**: Filtering bad rows, skipping already processed files.
- **Automation**: Skipping failed tasks in batch jobs while continuing others.
- **Logging/Monitoring**: Skip noisy entries while processing logs.

Example:
```python
for record in incoming_records:
    if record.is_duplicate or not is_valid(record):
        continue
    process_record(record)
    save_successfully_processed(record)
```

---

## 9. Machine Learning Usage

- **Data Preprocessing**: Skip corrupted samples or outliers during dataset iteration.
- **Training Loops**: Skip batches with NaN values or skip epochs based on conditions.
- **Evaluation**: Continue to next sample on prediction errors.
- **RAG / LLM pipelines**: Skip documents that don’t meet relevance threshold.
- **PyTorch DataLoader**: Custom collate functions often use similar skip patterns internally.

```python
for batch in dataloader:
    if torch.isnan(batch.labels).any():
        continue
    optimizer.zero_grad()
    ...
```

## 10. Resume Intelligence Project Integration

**Feature to add**: Robust resume filtering and cleaning pipeline.

**Why it belongs**: Real resumes come with noise — corrupted PDFs, duplicates, irrelevant formats, or missing key sections. `continue` lets you skip bad ones efficiently while keeping the main processing loop readable.

**Example implementation idea**:

```python
def process_resumes(resumes):
    successful = []
    for resume in resumes:
        if not resume.is_parsable:
            log_skip(resume, "corrupted")
            continue
        if not has_required_sections(resume):
            continue
        try:
            features = extract_features(resume)
        except Exception as e:
            log_skip(resume, str(e))
            continue
            
        successful.append(features)
    return successful
```

This pattern integrates well with `range()` batching, nested section loops, and `break` for early stopping.

## 11. Complexity Analysis

- `continue` itself is O(1).
- It improves average-case performance by avoiding unnecessary work in skipped iterations.
- Does not change worst-case asymptotic complexity.
