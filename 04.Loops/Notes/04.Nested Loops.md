**Nested Loops**

## 1. Introduction

Nested loops are loops inside other loops, allowing iteration over multiple dimensions or combinations of data. Python supports nesting any loop type (`for` or `while`). They exist because real-world data and computations often involve multiple levels of structure. For ML engineers, nested loops are essential for processing multi-dimensional data, grid searches, matrix operations, batch handling, and feature engineering.

## 2. Core Theory

- **Definition**: A loop structure where one or more loops are placed inside another.
- **Syntax**:
  ```python
  for outer in outer_iterable:
      for inner in inner_iterable:
          # code block
  ```
- **Key Properties**:
  - Total iterations = product of individual loop sizes.
  - Inner loops execute fully for each outer iteration.
  - Supports `break` and `continue` at each level; use labels (via flags) for complex control.

## 3. Core Methods / Operations

**Basic nested for loops**

```python
for i in range(3):
    for j in range(2):
        print(f"({i}, {j})")
```

**With different iterables**

```python
users = ['A', 'B']
metrics = ['acc', 'loss']
for user in users:
    for metric in metrics:
        print(user, metric)
```

**Early termination patterns**

```python
found = False
for i in range(10):
    for j in range(10):
        if condition:
            found = True
            break
    if found:
        break
```

## 4. Production Use Cases

1. **Data Processing**: Iterate over datasets and features for cleaning or transformation.
2. **Automation Scripts**: Process multi-level directory structures or configuration matrices.
3. **Backend APIs**: Generate combinations for batch requests or report generation.
4. **Machine Learning**: Hyperparameter grid search, confusion matrix computation, cross-validation folds.
5. **Model Evaluation**: Loop over multiple models and multiple test datasets.

## 5. Machine Learning Connection

- **NumPy / Pandas**: Used with `df.iterrows()` + inner feature loops, or manual grid search before `GridSearchCV`.
- **Scikit-learn**: Nested cross-validation, iterating over parameter grids.
- **PyTorch / TensorFlow**: Looping over epochs and batches, or over layers and neurons for custom initialization.
- **Feature Engineering**: Iterate over feature pairs for interaction terms.
- **Evaluation**: Compute metrics across multiple classes or multiple runs.

## 6. Common Mistakes

**Mistake 1**: High time complexity from large nested ranges.

```python
# Wrong for large N
for i in range(1000):
    for j in range(1000): ...
```

**Correct**: Use vectorized operations (NumPy) or reduce dimensions.

**Mistake 2**: Modifying outer loop variables from inner loop unintentionally.

**Mistake 3**: Deep nesting (>3 levels) making code hard to read.

**Mistake 4**: Using nested loops when `itertools.product()` is cleaner.

**Mistake 5**: Forgetting to reset inner state (e.g., counters) for each outer iteration.

**Best Practice**: Keep nesting shallow; refactor to functions or use comprehensions/`itertools` when possible.

## 7. Practical Examples

**1. Basic Example**
```python
matrix = [[1, 2], [3, 4]]
for row in matrix:
    for val in row:
        print(val, end=' ')
    print()
# Output: 1 2 \n 3 4
```

**2. Intermediate Example** – Feature combination
```python
features = ['age', 'salary']
for i in range(len(features)):
    for j in range(i+1, len(features)):
        print(f"Interaction: {features[i]} x {features[j]}")
```

**3. Logic Building Example** – Search with early exit
```python
data = [[10, 20], [30, 25], [40, 5]]
target = 25
found = False
for i, row in enumerate(data):
    for j, val in enumerate(row):
        if val == target:
            print(f"Found at position ({i}, {j})")
            found = True
            break
    if found:
        break
```

**4. Real Software Example** – Batch processing with metadata
```python
datasets = ['train', 'val', 'test']
batch_sizes = [32, 64]
for dataset in datasets:
    for bs in batch_sizes:
        print(f"Training {dataset} with batch_size={bs}")
```

**5. Production / ML Example** – Simple hyperparameter grid search
```python
learning_rates = [0.001, 0.01]
epochs_list = [10, 20]

results = []
for lr in learning_rates:
    for epochs in epochs_list:
        # Simulate training
        final_loss = 0.5 / (lr * epochs)  # mock
        results.append((lr, epochs, final_loss))
        print(f"lr={lr}, epochs={epochs} -> loss={final_loss:.4f}")
```

## 8. Time Complexity

| Operation                  | Time Complexity |
|---------------------------|-----------------|
| Two nested loops (n × m)  | O(n × m)        |
| Three nested loops        | O(n × m × p)    |

## 9. Key Takeaways

- Nested loops are powerful for multi-dimensional iteration but can become performance bottlenecks.
- Prefer `for` loops over `while` for nested structures when bounds are known.
- Use `enumerate()`, `zip()`, and `itertools.product()` to write cleaner nested logic.
- In ML, nested loops appear frequently in grid searches, data augmentation, and evaluation.
- Vectorize with NumPy/Pandas whenever possible to avoid slow Python loops.
- Keep nesting shallow and extract inner logic into functions for readability.
- Always analyze time complexity before using deep nested loops on large data.
- Master nested loops for custom ML workflows while knowing when to replace them with optimized libraries.
