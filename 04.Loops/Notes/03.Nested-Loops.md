# **Nested Loops**

## 1. Introduction

**Nested loops** occur when one loop is placed inside another. The inner loop runs completely for each iteration of the outer loop.

Python supports nesting any combination of `for` and `while` loops (and list comprehensions). It exists because many real-world problems involve **multiple dimensions** or hierarchical data — processing every combination of items, traversing grids/matrices, or handling sections within documents.

It solves problems requiring Cartesian products or multi-level iteration, but comes with a performance cost that must be managed carefully in production.

---

## 2. Intuition

Imagine reviewing resumes for multiple job positions:  
**For each job** (outer loop), **for each resume** in your database (inner loop), compute a match score.  

The outer loop sets the context, and the inner loop does detailed work for every item in that context.

---

## 3. Syntax

```python
# Basic nested for loops
for outer_item in outer_iterable:
    for inner_item in inner_iterable:
        # work with combination
        pass

# Mixed for + while
for row in matrix:
    col = 0
    while col < len(row):
        ...
        col += 1

# With else clauses
for i in range(3):
    for j in range(3):
        ...
    else:
        # runs after inner loop finishes
        ...
```

**Modern Python alternatives** (often preferred):
- List/dict/set comprehensions with multiple `for`
- `itertools.product()` for Cartesian products
- `zip()`, `enumerate()` in nested contexts

---

## 4. Internal Working

Each level maintains its own iterator state.  
For every single iteration of the outer loop, the inner loop runs from start to finish (or until `break`).

**Memory & Performance**: The total iterations = product of the sizes (O(n × m)). Deep nesting (3+ levels) quickly becomes expensive.

Python executes this naively — no automatic vectorization. That’s why we often replace deep nested loops with NumPy/Pandas vectorized operations or generators in production.

---

## 5. Built-in Methods / Operations

Key helpers for nested loops:

- `itertools.product(*iterables)`: Clean Cartesian product (better than manual nesting).
- `enumerate()` at multiple levels.
- `zip()` for parallel nested iteration.
- `itertools.chain.from_iterable()` for flattening.

**Practical example**:
```python
import itertools

skills = ["Python", "ML"]
jobs = ["Data Scientist", "ML Engineer"]

for job, skill in itertools.product(jobs, skills):
    print(f"Checking {skill} for {job}")
```

---

## 6. Common Mistakes

- **Performance blowup**: Nested loops over large datasets without early termination or optimization.
- Modifying outer collection from inner loop.
- Deep nesting (4+ levels) making code unreadable.
- Using nested loops when a single loop + data structure (dict of dicts, etc.) would suffice.
- Forgetting to reset inner loop variables.

**Wrong**:
```python
for i in range(n):
    for j in range(m):
        result[i][j] = heavy_computation()  # Slow if heavy_computation is expensive
```

---

## 7. Best Practices

- Keep nesting shallow (max 2–3 levels). Extract inner logic to functions.
- Use `break` / `continue` with labels simulated via flags when needed.
- Prefer `itertools.product()`, comprehensions, or vectorized libraries.
- Profile nested loops — they are common performance bottlenecks.
- Use meaningful variable names: `for resume in resumes: for section in resume.sections:`

---

## 8. Production Usage

- **Backend**: Cross-referencing data (users × permissions, orders × items).
- **Data Engineering**: Joining datasets, processing multi-level JSON/XML.
- **ETL**: Nested batch processing (files → documents → chunks).
- **APIs**: Rate-limited calls with retries inside pagination loops.

Example:
```python
for batch in batches:
    for record in batch:
        retry_process(record)
```

---

## 9. Machine Learning Usage

- **Data Preprocessing**: Nested loops over epochs × batches (though DataLoader hides it).
- **Matrix Operations**: Before NumPy, people used nested loops for dot products (now avoid).
- **Scikit-learn / Evaluation**: Cross-validation (folds × models).
- **PyTorch / TensorFlow**: Custom training with nested loops over layers or attention heads.
- **RAG / LLM**: For document in documents: for chunk in split(document): embed(chunk).

**Modern approach**:
```python
# Instead of nested loops:
import numpy as np
similarity_matrix = np.dot(resume_embeddings, job_embeddings.T)
```

## 11. Complexity Analysis

- **Time**: O(n × m) for two-level nesting (n = outer size, m = inner size).
- **Space**: Usually O(1) extra (excluding output). Can be O(n) if storing results.
- **Deep nesting**: Quickly reaches O(n^k) — very dangerous at scale.

**Rule of thumb**: If total iterations > 10^6–10^7, consider optimization or vectorization.

## 14. Code Review Checklist

- **Avoid**: Deep nesting (>3 levels), unnecessary O(n²) when better algorithms exist, poor variable names.
- **Prefer**: Extract inner loops to functions, use `itertools` or comprehensions when cleaner, vectorized libraries for numeric work.
- **Naming**: `for job in jobs: for resume in resumes:`, `for section in resume.sections:`
- **Interviewers expect**: Awareness of performance implications, ability to optimize or replace with better approaches (hash maps, vectorization, etc.).
